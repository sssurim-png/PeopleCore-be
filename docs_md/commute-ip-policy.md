# 출퇴근 IP 기반 근무지 검증 — CIDR 매칭 + XFF 정규화

> 회사가 등록한 공인 IP 대역에서만 체크인을 허용하는 정책을, **멀티 홉 인프라(NLB → Ingress → API Gateway → hr-service)** 에서도 정확히 동작하도록 단일 진입점(`ClientIpExtractor`)으로 정리한 설계 문서입니다. 등록 모달과 검증 로직이 같은 IP 를 보도록 강제해 "내가 등록한 IP 인데 매칭 실패" 무한 루프 사고를 차단합니다.

## 1. 개요

| 항목 | 내용 |
|------|------|
| 정책 | 회사가 등록한 공인 IP **CIDR 대역** 에서만 체크인 허용 |
| IP 추출 진입점 | `ClientIpExtractor` — **단일 클래스로 일원화** |
| 헤더 우선순위 | `X-Forwarded-For` 순회 → loopback 스킵 → IPv4-매핑 IPv6 정규화 |

| 검증/표시 일관성 | 등록 모달의 "내 현재 IP" 표시와 출퇴근 검증이 **동일 로직** |

## 2. 배경

### 문제 사항
- 출퇴근 IP 정책: 회사가 등록한 공인 IP 대역에서만 체크인 허용
- NLB/Ingress 거쳐 사설 IP 만 도달 시 CIDR 매칭 전부 실패 → **전사 출퇴근 차단 사고 가능**
- "내 현재 IP" 표시(등록 모달)와 출퇴근 검증이 서로 다른 IP 추출 로직을 쓰면, 사용자가 등록한 IP 가 매칭에서 누락되는 **무한 루프 사고**

### 원인 분석
- 멀티 홉 환경(NLB → Ingress → API Gateway → hr-service)에서 `request.getRemoteAddr()` 는 **직전 홉의 사설 IP 만** 노출
- 클라이언트 공인 IP 는 `X-Forwarded-For` 헤더에 누적되지만, 환경별 형태가 상이
  - IPv4-매핑 IPv6 (`::ffff:1.2.3.4`)
  - loopback 토큰 (`::1`, `127.0.0.1`) 이 앞에 혼입
- 단일 정규화 부재 시 매칭 산발 실패

## 3. 후보 비교

| 옵션 | 장점 | 단점 | 채택 |
|------|------|------|------|
| `getRemoteAddr()` 단독 | 단순 | NLB 뒤에서는 사설 IP 만 도달, 정책 무력화 | ❌ |
| XFF 첫 토큰만 사용 | 구현 간단 | loopback 토큰이 앞에 끼면 오판정, 인프라 케이스에 취약 | ❌ |
| 외부 라이브러리(`ipaddress`) 의존 | 정규화 일관성 보장 | 의존성 추가 + 검증/매칭 로직 분산 | ❌ |
| `ClientIpExtractor` 단일 진입점 + 자체 CIDR 매칭 | 일관성 + 의존성 0 | 케이스 직접 처리 | ✅ |

## 4. 해결 설계

### 4.1 IP 추출 — `ClientIpExtractor`

| 순서 | 처리 | 비고 |
|------|------|------|
| 1 | `X-Forwarded-For` 콤마 분리 후 앞에서부터 순회 | 빈 토큰 / `unknown` 스킵 |
| 2 | loopback 토큰(`::1`, `127.0.0.1`) 스킵 | 앞 홉 인입 케이스 방어 |
| 3 | 첫 유효 토큰 채택 | — |
| 4 | IPv4-매핑 IPv6(`::ffff:1.2.3.4`) → IPv4 정규화 | 비교 단위 통일 |
| 5 | XFF 없으면 `request.getRemoteAddr()` 폴백 | 단일 홉 환경 대응 |

- **출퇴근 검증과 등록 모달이 동일 IP 를 보도록 강제**
- 환경별 헤더 형태(IPv6/loopback/매핑)를 한 곳에서 정규화

### 4.2 CIDR 매칭 — `InetAddress` + 비트연산

- JDK 표준 `InetAddress.getAddress()` byte[] 비교 — 외부 의존성 0
- IPv4/IPv6 모두 byte 단위로 일관 처리
- `prefix = 0` (전체 허용) edge case 명시 처리

### 4.3 운영 가이드

| 항목 | 가이드 |
|------|--------|
| 인프라 설정 | NLB / Ingress / API Gateway 가 클라이언트 IP 를 XFF 에 보존하도록 구성 |
| 배포 직후 검증 | 디버그 로그로 추출된 IP 가 클라이언트 공인 IP 와 일치하는지 확인 |
| 등록 UX | 등록 모달의 "내 현재 IP" 는 검증과 동일 메서드를 호출 → 등록 직후 즉시 매칭 보장 |
