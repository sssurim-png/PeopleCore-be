# 사원 등급 시계열 변동성 진단 기준

## 분석 정의

**사원(employeeId)** 단위로 시즌별 **finalGrade** 시계열을 추적하여
사원의 등급 변동 패턴을 분류:

- 등급 = S(5) / A(4) / B(3) / C(2) / D(1) 점수 매핑
- 시즌 시퀀스 → 변동 패턴 라벨
- 활동 시즌 수에 따라 분석 깊이 차등 (2시즌 / 3+시즌)

> **왜 finalGrade 인가?**
> 사원 입장에서 실제 부여받은 최종 등급이 finalGrade. 본 분석은
> "사원 등급이 시간에 따라 어떻게 움직였나" 를 보는 것이므로
> 보정 단계가 아닌 **확정된 결과** 를 추적한다.
> (평가자 본인 패턴은 #5 — managerGrade 사용)

## 핵심 지표 (수치)

| 지표 | 정의 | 단위 |
|------|------|------|
| `grade_sequence` | 시즌별 등급 배열 (예: ["S","A","S","A"]) | - |
| `n_seasons_present` | 사원이 평가받은 시즌 수 | 회 |
| `mean_grade_score` | 시즌 등급 점수 평균 | 점 (1~5) |
| `stdev_grade` | 시즌 등급 점수 표준편차 (변동 정도) | 점 |
| `trend_slope` | 등급 점수의 시간 회귀 기울기 (3+시즌만) | 점/시즌 |
| `direction_label` | 2시즌 사원의 방향 (유지/상승/하락) | - |

## 매개변수

| 매개변수 | 기본값 | 의미 |
|---------|-------|------|
| `min_seasons` | 2 | 분석 대상 최소 활동 시즌 수 |
| `lookback_seasons` | 4 | 최대 추적 시즌 수 (실제 데이터 보유 한도) |
| `stable_stdev_threshold` | 0.5 | 안정 판정 표준편차 임계치 (3+시즌) |
| `trend_slope_threshold` | 0.3 | 상승/하락 판정 회귀 기울기 임계치 (3+시즌) |
| `high_grade_score` | 4 | "우수" 판정 평균 점수 임계치 (A 이상) |
| `low_grade_score` | 2 | "하위" 판정 평균 점수 임계치 (C 이하) |

## 분석 흐름

### 1단계: 사원별 등급 시퀀스 수집
- 사원 ID 별로 시즌별 finalGrade 추출 → `grade_sequence`
- 점수 매핑: S=5, A=4, B=3, C=2, D=1
- 활동 시즌 수 < `min_seasons` 인 사원 제외

### 2단계: 시즌 수에 따라 분석 분기

#### 2-A. 2시즌 사원 — 단순 비교
- 변화 없음 → **유지**
- 한 단계 이상 올라감 → **상승**
- 한 단계 이상 내려감 → **하락**
- 평균 점수에 따라 우수/평균/하위 추가 라벨

#### 2-B. 3+시즌 사원 — 풀 패턴 분석
- `stdev_grade`, `trend_slope` 계산
- 패턴 라벨 부여 (아래 표)

### 3단계: 패턴 라벨 부여

## 패턴 라벨

### 3+시즌 사원 (풀 패턴)

| 패턴 | 조건 | 정의 |
|------|------|------|
| **안정 우수** | `stdev_grade ≤ 0.5` AND `mean_grade_score ≥ 4` | A 이상 일관 유지 |
| **안정 평균** | `stdev_grade ≤ 0.5` AND `2 < mean_grade_score < 4` | B 일관 유지 |
| **안정 하위** | `stdev_grade ≤ 0.5` AND `mean_grade_score ≤ 2` | C 이하 일관 유지 |
| **상승** | `trend_slope ≥ 0.3` AND `stdev_grade > 0.5` | 시간 흐름에 따라 등급 상승 |
| **하락** | `trend_slope ≤ -0.3` AND `stdev_grade > 0.5` | 시간 흐름에 따라 등급 하락 |
| **변동** | `stdev_grade > 0.5` AND `|trend_slope| < 0.3` | 일관 방향 X, 들쭉날쭉 |

### 2시즌 사원 (단순 비교)

| 패턴 | 조건 | 정의 |
|------|------|------|
| **유지 (우수/평균/하위)** | 두 시즌 등급 동일 | 등급 변화 없음 + 평균 점수 라벨 |
| **상승** | 두 번째 시즌 점수 > 첫 시즌 점수 | 한 단계 이상 올라감 |
| **하락** | 두 번째 시즌 점수 < 첫 시즌 점수 | 한 단계 이상 내려감 |

> 2시즌 데이터로는 "변동" 판정 불가 (1회 변경은 추세인지 일시인지 구분 X)

## 시그널 의미 — 패턴별 가치 + 결합 분석

| 패턴 | 시그널 | 결합 분석 권고 |
|------|-------|---------------|
| **안정 우수** | 우수인재 후보 | + #2 (보상 누락) 결합 시 보상 검토 우선순위 ↑ |
| **안정 평균** | 안정 기여자 | HR 검토 |
| **안정 하위** | 부진 누적 | + #9 와 결합 (PIP 후보) |
| **상승** | 성장 인재 | 육성·승진 검토 자료 |
| **하락** | 등급 하락 추세 | + #2 (보상 누락) 결합 시 HR 검토 우선순위 ↑ |
| **변동** | 등급 변동 큼 OR 평가 일관성 X | + #5 (평가자 편향) → 평가 일관성 의심 분리 |

→ 모든 패턴은 **사원 본인 역량 단정 X**, 데이터적 관찰일 뿐.

## 권고 액션

1. 분석 결과 = 사원 등급 변동 현황 자료 (즉시 처분 X)
2. 패턴별 후보 그룹 도출 → 결합 분석으로 강도 판정
3. 우선 결합 권고:
   - 안정 우수 → #2 결합 → 보상 위치 확인
   - 변동 → #5 결합 → 평가자 패턴 영향 분리
   - 안정 하위 → #9 결합 → 부진 누적 사원 식별
4. 결정은 권한자의 판단

## 표현 가이드 — 본문 vs AI 박스

본문 (수치만):
- "분석 대상 사원 380명 (2시즌 이상 재직)"
- "  - 3+시즌 사원: 220명 / 2시즌 사원: 160명"
- "사원 EMP-021: [S, S, A, S] (4시즌, mean 4.75, stdev 0.43, slope -0.10) → 안정 우수"
- "사원 EMP-105: [B, A, S] (3시즌, mean 4.0, stdev 0.82, slope +0.50) → 상승"
- "사원 EMP-208: [A, B] (2시즌, mean 3.5) → 하락"

AI 참고 박스 (1-2줄):
- 안정 우수:
  > "EMP-021 은 4시즌 연속 A 이상 등급을 유지하고 있어 안정 우수 인재 후보입니다.
  > #2 (보상 누락) 결합 분석으로 보상 위치 점검을 권고합니다."
- 변동:
  > "EMP-150 의 등급 변동이 큽니다. 사원 본인 성과 변동 가능성과 평가자 패턴 영향 가능성이
  > 모두 존재하므로 #5 (평가자 편향) 결합 분석을 권고합니다."

## 분석 도구

- `find_employee_grade_volatility(min_seasons, lookback_seasons, stable_stdev_threshold, trend_slope_threshold)`

## 한계

1. **등급 자체의 정확성 가정** — 등급이 사원 실제 성과를 정확히 반영했다는 가정. 평가 일관성 의문은 #5 별도 분석
2. **2시즌 사원은 추세 판정 약함** — 1회 변경이 추세인지 일시 변동인지 구분 X
3. **3시즌 사원도 분산 큼** — 표본 시즌 수가 적으면 stdev·slope 신뢰도 제한적
4. **부서 이동·역할 변경 미보정** — 다른 직무로 옮겨서 등급이 변한 경우도 변동으로 잡힘
5. **finalGrade 의존** — 보정 누적 결과 사용. 보정 단계별 분석은 별도
6. **신규 입사자 제외** — 1시즌만 활동한 사원은 분석 X (`min_seasons` 미만)
7. **회사 운영 시즌 한도** — 본 시스템은 3~4시즌 데이터만 보유 → 장기 추세 분석 X
