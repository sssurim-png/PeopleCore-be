# 보상-성과 정합성 진단 기준

## 분석 정의

직급(rankId) 단위로 보상 체계가 성과(등급)를 얼마나 정합적으로 반영하는지
**3개 세부 진단** 으로 측정하고, 마지막에 **부당 보상 후보 사원** 도출:

- **세부 1 — 단기 변별력**: 단년 등급 ↔ 성과급(BONUS)
- **세부 2 — 장기 변별력**: 누적 등급 ↔ 기본급(SALARY) 시즌간 인상률
- **세부 3 — 부당 보상 후보**: 재직·근태 양호 + 진급 누락 + 저연봉 + 성과 조건 (두 케이스 OR)
  - **케이스 A (통상자)**: B 이상 + 통상 진급 년수 + 버퍼 초과
  - **케이스 B (우수자)**: A 이상 + 통상 진급 년수 도달

> **부서별 분석 X** — 직급 단위만. 부서별 분포는 #4 참조.

## 데이터 소스

| 출처 | 활용 필드 | 분석 용도 |
|------|---------|----------|
| 평가 (`finalGrade`) | 시즌별 등급 (S/A/B/C/D) | 단년·다년 등급 |
| `PayrollDetails` | `payItemCategory = SALARY` 합계 | 기본급 |
| `PayrollDetails` | `payItemCategory = BONUS` 합계 | 성과급 |
| `Employee.hireDate` | 입사일 | 재직년수 |
| `HrOrder.orderType = PROMOTION` | 진급 발령 이력 | 진급 누락 판정 |
| `CommuteRecord.workStatus` | LATE/ABSENT 빈도 | 근태 양호 판정 |
| `Title.rankId` | 직급 | 직급 단위 풀 |

## 핵심 지표

### 세부 1: 단기 변별력 (직급 단위)

| 지표 | 정의 |
|------|------|
| `corr_grade_bonus` | 시즌 등급 점수(S=5..D=1) vs 시즌 성과급(BONUS) 합계 피어슨 상관 |
| `mean_bonus_by_grade` | 직급 내 등급별 평균 성과급 |

### 세부 2: 장기 변별력 (직급 단위)

| 지표 | 정의 |
|------|------|
| `corr_avg_grade_salary_growth` | 다년 등급 평균 vs 시즌간 기본급 인상률(%) 피어슨 상관 |
| `mean_salary_growth_company` | 사내 평균 기본급 인상률 (통상 비교 기준) |
| `mean_salary_growth_top_grade` | 등급 평균 A 이상 사원의 기본급 인상률 |

### 세부 3: 부당 보상 후보 (사원 단위)

| 지표 | 정의 |
|------|------|
| `tenure_years` | 재직년수 (현재 - hireDate) |
| `years_since_last_promotion` | 마지막 진급 이후 경과 년수 (또는 입사 이후 진급 0회면 재직년수와 동일) |
| `attendance_score_pctile` | 사내 근태 양호도 백분위 (LATE+ABSENT 비율 역수 기준) |
| `avg_grade_score` | 분석 시즌 등급 평균 점수 (S=5..D=1) |
| `min_grade_in_range` | 분석 시즌 중 최저 등급 |
| `salary_pctile_in_rank` | 직급 내 본인 연봉 백분위 |

## 분석 흐름

### 0단계: 직급별 표본 크기 자동 모드 전환

각 직급의 분석 대상 사원 수에 따라 모드 자동 결정:

| 직급 인원 | 모드 | 분석 |
|---------|-----|------|
| ≥ 20 | **FULL** | 피어슨 상관 안정 — 정상 분석 |
| 10~19 | **PARTIAL** | 분석은 진행하나 "신뢰도 제한적" 명시 |
| 5~9 | **WEAK** | 참고 자료 수준, 단정 X |
| < 5 | **INSUFFICIENT** | 분석 스킵 + 사실 명시 ("표본 부족") |

각 등급별로도 표본 < 3이면 그 등급 평균 분석 제외 (단, 직급 전체 분석은 진행 가능).

### 1단계: 직급 단위 변별력 진단 (세부 1, 2)

```
각 직급(rankId) 풀에 대해 (모드별 분석):
  - 시즌 등급 vs 성과급 → corr_grade_bonus
  - 다년 등급 평균 vs 기본급 인상률 → corr_avg_grade_salary_growth
  - 직급 라벨 부여 (모드 함께 표시)
  - INSUFFICIENT 모드는 "표본 부족" 라벨 + 분석 스킵
```

### 2단계: 부당 보상 후보 발굴 (세부 3)

```
모든 사원에 대해 공통 3개 조건 + (케이스 A OR 케이스 B):

[공통 조건]
  - tenure_years ≥ min_tenure_years (5년)        (재직 안정)
  - attendance_score_pctile ≥ 70                 (근태 양호)
  - salary_pctile_in_rank ≤ 30                   (직급 내 저연봉)

[케이스 A — 통상자: 진급 년수 + 버퍼 초과]
  - years_since_last_promotion ≥ (typical_promotion_years + overdue_buffer_years)
                                                  → 기본 4 + 2 = 6년 정체
  - avg_grade_score ≥ 3.0                         (B 이상 평균)
  - min_grade_in_range ≥ "B"                      (모든 시즌 B 이상)

[케이스 B — 우수자: 진급 년수 도달]
  - years_since_last_promotion ≥ typical_promotion_years
                                                  → 기본 4년 정체
  - avg_grade_score ≥ 4.0                         (A 이상 평균)
  - min_grade_in_range ≥ "A"                      (모든 시즌 A 이상)

→ 공통 조건 모두 AND (케이스 A OR 케이스 B) 충족 시 후보

두 케이스의 의미:
  - 우수자(A 이상)는 통상 진급 년수만 지나도 부당 — 더 빨리 진급해야 마땅
  - 통상자(B 이상)는 통상 + 버퍼를 지나야 부당 — 일반 사원의 자연스러운 정체 인정
  - 성과 부진(D/C 발생)으로 정체된 사원은 양쪽 케이스 모두 미해당 → 후보 제외
```

## 사내 통상 비교 — 동적 임계치

본 시스템은 절대 임계치 대신 사내 통상값을 사용:

- **세부 1, 2 변별력 상관**은 절대값 임계치 (피어슨 상관 0.3, 0.5) — 통계 관행
- **세부 3 후보 조건**은 사내 통상 백분위 (근태 상위 30%, 직급 내 연봉 하위 30%)
- 외부 권고치 인용 X, 사내 데이터 자체가 기준

## 직급 라벨 (세부 1·2 결합)

| 라벨 | 조건 |
|------|------|
| **변별력 정상** | 단기 OR 장기 상관 ≥ 0.5 |
| **단기만 작동** | 단기 상관 ≥ 0.5 AND 장기 상관 < 0.3 (성과급은 차등, 기본급은 일률) |
| **장기만 작동** | 단기 상관 < 0.3 AND 장기 상관 ≥ 0.5 (드문 케이스) |
| **변별력 약함** | 단기·장기 모두 상관 < 0.3 |
| **신뢰도 제한적** | 직급 인원 5~19 (PARTIAL/WEAK 모드) — 위 라벨 부여하되 신뢰도 한계 명시 |
| **표본 부족** | 직급 인원 < 5 (INSUFFICIENT) — 분석 스킵, 사실 명시 |

## 시그널 의미

### 직급 변별력 (세부 1·2)

| 라벨 | 가능성 1 | 가능성 2 |
|------|---------|---------|
| 변별력 약함 | 보상 체계가 등급 반영 X | 등급 자체 신뢰도 의문 (#5 결합 검토) |
| 단기만 작동 | 정상 작동 (단기는 성과급, 장기는 기본급으로 분리) | 기본급 인상이 등급 무시하는 구조 |
| 장기만 작동 | 성과급 체계 변별력 부족 | 데이터 우연 (드문 케이스) |

### 부당 보상 후보 (세부 3)

가능성 1: 진급·보상 누락 (구조적 문제)
가능성 2: 본인의 직급 잔류 의향, 외부 평가 사정 (개별 정성 정보 필요)

→ 본 분석으로 단정 X, **검토 후보 명단** 제공

## 권고 액션

1. 분석 결과 = 변별력 진단 + 부당 후보 명단 (즉시 처분 X)
2. 직급 라벨별 후속:
   - 변별력 약함 → 보상 체계 점검 안건 (HR + 재무)
   - 단기만 작동 → 정상 또는 기본급 인상 정책 점검
3. 부당 후보 → **개별 면담 + 본인 의향 확인 + 진급/보상 검토**
4. 결합 분석 권고:
   - + #2 (보상 누락) → 등급 우수 후보와 안정 기여자 후보 동시 검토
   - + #5 (평가자 편향) → 변별력 약함 직급의 등급 신뢰도 점검
   - + #6 (등급 변동) → 안정 우수 사원이 부당 후보에 들어있는지 교차
5. 결정은 권한자의 판단

## 한계

1. **연봉 = 누적 결과** — 입사 시 결정 + 매년 인상 누적이라 단년 등급만으로 차등 어려움. 세부 2(장기 변별력)로 일부 보완하나 완전 분리 X
2. **직무 차이 미보정** — 같은 직급이라도 R&D/영업/지원 직무 따라 보상 다름
3. **경력 입사 변동** — 중간 입사자는 입사 시점 연봉이 직급 내 분포와 일치하지 않을 수 있음
4. **진급 누락의 사정 미반영** — 본인이 직급 잔류를 원했거나 외부 평가 사정이 있을 수 있음
5. **ALLOWANCE(수당) 제외** — 직무·근태 영향 큰 수당은 등급 변별력 진단에 노이즈 → 본 분석에서 제외 (분석 도구 시그니처에서 SALARY+BONUS 만 사용)
6. **finalGrade 의존** — 보정 누적 결과 사용. 평가 자체 신뢰도 의문은 #5 별도
7. **근태 양호 정의** — LATE/ABSENT 비율의 역백분위 사용. 다른 근태 시그널(연차 사용·인터미션)은 #7 영역
8. **통상 진급 년수 가정** — 한국 일반 4년 기준. 회사 인사 규정에 명문 진급 주기 있으면 별도 매개변수로 재조정 필요. 직급별로 다르면 dict 형태로 입력
9. **직급 표본 크기 의존** — 직급 인원 < 5 면 분석 자체 스킵. 5~9명 WEAK / 10~19 PARTIAL 모드는 신뢰도 제한적이라 "참고 자료" 수준. 20명+ 부터 피어슨 상관 안정. 분석 결과 활용 시 모드 명시 필수
