# 부서별 워라밸 진단 기준

## 분석 정의

근태(`CommuteRecord`) + 휴가(`VacationBalance`) 데이터로
부서별 업무 부담을 사내 통상값과 비교하여 진단:

- 부서별 야근·휴일근무·야간근무·연차사용률 집계
- 사내 통상 백분위로 분류 → 과부하/통상/여유 라벨
- 부서 진단 + 부서 내 사원 단위 과부하 후보 도출 (선택)

## 데이터 소스

| 출처 | 활용 필드 | 분석 용도 |
|------|---------|----------|
| `CommuteRecord.overtimeMinutes` | 총 초과근무 분 | 야근 수준 |
| `CommuteRecord.unrecognizedOtMinutes` | 미인정 초과 분 | 무급/자발적 야근 추정 |
| `CommuteRecord.recognizedNightMinutes` | 야간근무 분 (22:00~06:00) | 심야 부담 |
| `CommuteRecord.recognizedHolidayMinutes` | 휴일근무 분 | 휴일 부담 |
| `CommuteRecord.workStatus` | NORMAL/LATE/EARLY_LEAVE/HOLIDAY_WORK/ABSENT | 출근 패턴 |
| `VacationBalance.usedDays / totalDays` | 연차 사용률 | 휴식 보장도 |
| `VacationBalance.expiredDays / totalDays` | 연차 만료 비율 | 미사용 만료 신호 |

## 핵심 지표 (수치)

### 부서 단위

| 지표 | 정의 | 단위 |
|------|------|------|
| `mean_overtime_min_per_emp` | 부서 사원 1인당 월평균 야근 분 | 분/월 |
| `unrecognized_ot_ratio` | 미인정 초과 분 ÷ 전체 초과 분 | % |
| `mean_night_min_per_emp` | 1인당 월평균 야간근무 분 | 분/월 |
| `holiday_work_ratio` | 휴일근무 발생 사원 비율 | % |
| `late_day_ratio` | 평균 지각일수 비율 | % |
| `mean_leave_usage_rate` | 부서 평균 연차 사용률 (annual 한정) | % |
| `expired_leave_ratio` | 부서 평균 연차 만료 비율 | % |

### 사원 단위 (부서 내 후보 도출용)

| 지표 | 정의 |
|------|------|
| `emp_overtime_min` | 사원 본인 월 야근 분 |
| `emp_overtime_pctile_in_dept` | 부서 내 본인 야근 백분위 |
| `emp_leave_usage_rate` | 사원 본인 연차 사용률 |

## 매개변수

| 매개변수 | 기본값 | 의미 |
|---------|-------|------|
| `min_dept_size` | 10 | 분석 최소 부서 인원 |
| `analysis_period_months` | 3 | 집계 기간 (월) |
| `dept_top_pctile` | 90 | 과부하 부서 판정 백분위 (전사 부서 풀) |
| `dept_bottom_pctile` | 10 | 여유 부서 판정 백분위 |
| `emp_top_pctile_in_dept` | 90 | 부서 내 과부하 사원 백분위 |
| `low_leave_usage_threshold` | 50 | 연차 사용률 낮음 절대 임계치 (%) |
| `high_unrecognized_ot_threshold` | 50 | 미인정 초과 비중 높음 절대 임계치 (%) |

## 분석 흐름

### 1단계: 부서별 집계
- `analysis_period_months` 기간의 근태·휴가 데이터 집계
- 부서별 핵심 지표 7종 계산

### 2단계: 부서 라벨 (사내 통상 백분위)
- 전사 부서 분포 안에서 본 부서 백분위 계산
- 야근시간 또는 휴일근무율 또는 야간근무 백분위 ≥ 90 → **과부하 부서**
- 연차 사용률 백분위 ≤ 10 또는 절대값 ≤ 50% → **연차 미사용 부서**
- 미인정 초과 비중 ≥ 50% → **무급 야근 의심 부서**
- 위 모두 미해당 → **통상 범위**

### 3단계: 부서 내 사원 후보 도출 (선택)
- 부서 내 사원 야근 분 백분위 계산
- 본인 백분위 ≥ 90 → **부서 내 과부하 사원** 후보

## 사내 통상 비교 — 동적 임계치

본 시스템은 절대 임계치 대신 사내 통상값을 사용:

- 비교 기준 = **그 기간 전사 부서 풀 분포**
- 단, 일부 지표는 절대 임계치 사용 (해석이 명확한 경우):
  - `low_leave_usage_threshold = 50%` (사용률 50% 미만은 명백한 저활용)
  - `high_unrecognized_ot_threshold = 50%` (초과근무의 절반 이상이 미인정 = 정책상 명확한 신호)
- 외부 권고치 인용 X, 사내 데이터 자체가 기준

## 부서 라벨

| 라벨 | 조건 |
|------|------|
| **과부하 부서** | 야근·야간·휴일근무 중 1개 이상 백분위 ≥ 90 |
| **연차 미사용 부서** | 연차 사용률 백분위 ≤ 10 또는 절대 ≤ 50% |
| **무급 야근 의심 부서** | 미인정 초과 비중 ≥ 50% |
| **통상 범위** | 위 모두 미해당 |

> 한 부서가 여러 라벨 동시 부착 가능 (예: 과부하 + 연차 미사용).

## 시그널 의미 — 양쪽 가능성 명시

| 라벨 | 가능성 1 | 가능성 2 |
|------|---------|---------|
| 과부하 부서 | 강제 야근·구조적 부담 | 분기말·프로젝트 마감 등 일시적 사이클 |
| 연차 미사용 | 휴가 사용 어려운 분위기 | 사원 본인 의향 (연차 보존 선호) |
| 무급 야근 의심 | 정책 미준수 또는 인정 절차 문제 | 사원 자발적 잔여 업무 처리 |

→ **본 분석으로 자발성·강제성 단정 X.** 사업 사이클·조직 사정 정성 결합 권고.

## 권고 액션

1. 분석 결과 = 수치 부담 현황 자료 (즉시 처분 X)
2. 라벨 부서 도출 → HR 검토 안건 후보
3. 부서 내 과부하 사원 후보 별도 표시 → 개별 면담 자료
4. 결합 분석 권고 (후속 보고서 항목):
   - **+ 급여(연봉·수당)** → 야근 시간이 보상에 충분히 반영되는지 (별도 분석)
   - **+ #5 (평가자 편향)** → 과부하 부서가 평가까지 박하게 받는지
   - **+ #6 (등급 변동)** → 과부하 부서 사원 등급 추세
   - **+ #2 (보상 누락)** → 과부하 + 우수 + 보상 하위 결합 사원 (HR 검토 우선)
5. 결정은 권한자의 판단

## 표현 가이드 — 본문 vs AI 박스

본문 (수치만):
- "분석 기간 2026-02 ~ 2026-04 (3개월), 분석 부서 12개 (≥10명)"
- "전사 평균 야근 18.4 시간/월, 연차 사용률 평균 67%"
- "R&D팀 (n=85): 야근 32.5h/월 (pctile 95), 연차 사용률 41% (pctile 8), 미인정 초과 62%"
  → 라벨: 과부하 + 연차 미사용 + 무급 야근 의심
- "재무팀 (n=42): 야근 8.2h/월 (pctile 22), 연차 사용률 78% (pctile 70)"
  → 라벨: 통상 범위
- "R&D팀 내 과부하 사원: 12명 (부서 인원의 14%)"

AI 참고 박스 (1-2줄, 양쪽 가능성 명시):
- 과부하 부서:
  > "R&D팀은 전사 통상 대비 야근·야간·휴일 부담 모두 상위 백분위입니다.
  > 구조적 부담 가능성과 분기 사이클·프로젝트 일시 부담 가능성 모두 존재 →
  > 사업 사이클 정성 결합 + 야근의 보상 반영 여부 후속 분석을 권고합니다."
- 무급 야근 의심:
  > "R&D팀은 초과근무의 62% 가 미인정 분류입니다.
  > 정책 미준수 가능성과 자발적 잔여 업무 가능성 모두 존재 →
  > 인정 절차 점검과 사원 면담 결합 검토를 권고합니다."

## 분석 도구

- `analyze_workload_balance(analysis_period_months, min_dept_size, dept_top_pctile, low_leave_usage_threshold, high_unrecognized_ot_threshold)`

## 한계

1. **자발/강제 야근 구분 불가** — 데이터로 판단 X. `unrecognizedOtMinutes` 비중은 정책 신호일 뿐 자발성 단정 X
2. **직무·업무 사이클 차이 미보정** — R&D 마감기 vs 영업 분기말 등 본질적 사이클 차이 미반영
3. **시즌성 영향** — 분기말·연말 등 일시 부담을 평상 패턴으로 오해 가능. 분석 기간을 반기 이상으로 늘리면 일부 보완
4. **연차 미사용의 의도 분리 불가** — 본인 의향(연차 보존)과 분위기(눈치) 데이터로 구분 X
5. **재택·외근 데이터 정합성 의존** — 출퇴근 미체크·외부 근무 시 데이터 누락 가능
6. **휴가 유형 다양성** — `VacationType` 이 여러 종류. "연차 사용률" 은 `StatutoryVacationType.ANNUAL` 한정 (그 외 병가·경조사 등 별도)
7. **소규모 부서 노이즈** — `min_dept_size` 미만 제외, 임계치 인접 부서는 변동 큼
