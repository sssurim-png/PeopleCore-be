# 전사 진단 종합 보고서 기준

## 분석 정의

**#1~#9 의 결과를 5개 섹션으로 통합** 한 표준화된 종합 보고서:

- 한 호출로 사전 정의된 7개 분석 모두 실행
- 결과를 **전사·부서·사원** 차원에서 통합
- 형식 일관성 보장 (재현 가능, 분기 보고서 적합)
- 자유 쿼리(2~3개 짬뽕)는 본 도구 X — 개별 분석 도구 호출이 맞음

> **언제 호출되는가**
> - "전사 진단 보고서", "종합 분석", "전체 결과 정리해줘" 같은 명시적 종합 요청
> - 분기·반기 정기 보고용
> - 단발 질문 ("R&D 워라밸 어때?") 은 개별 분석 도구 호출

## 보고서 구조 (5개 섹션)

### §1. 전사 요약 (1페이지)

```
[전사 진단 종합 — 2025H2]

평가 변별력 (#1):    G3 정상 / G4 약함 / G5 표본 부족
부서 등급 분포 (#4): 상위 편중 1개 / 하위 편중 2개 / 통상 4개
평가자 패턴 (#5):    FALLBACK 모드 (풀 7명) — 지속 상위 1명
워라밸 (#7):         과부하 1개 / 무급야근 의심 1개 / 통상 5개
산업안전 (#9):       법규 위반 1개 부서 (24명) / 임계근접 1개

[AI 박스] 1-2줄 종합 권고
```

### §2. 부서별 매트릭스

7개 부서 × 5개 분석 라벨 표:

```
부서       #4 등급분포   #5 평가자(*)  #7 워라밸     #9 산업안전
개발팀     상위 편중     지속 상위    과부하        통상
재무팀     하위 편중     통상         통상          통상
영업팀     통상          통상         임계근접      위반
마케팅팀   통상          변동         통상          통상
...

(*) #5 는 평가자 단위 분석이라 부서장 평가 패턴 표시
```

### §3. 사원별 후보 통합 명단

`#1-3 부당 보상` + `#2 보상 누락` + `#6 등급변동` 후보 합집합:

```
사원         #2 누락   #1-3 부당     #6 등급변동      결합 강도
EMP-058      O        O (케A)      안정 평균        강함 (2개)
EMP-023      O        O (케B)      안정 우수        강함 (3개)
EMP-105      -        -            상승             단일
EMP-302      -        O (케B)      안정 우수        중간 (2개)
...

결합 강도 = 동시 도출된 분석 수
  강함(3+): 즉시 면담 우선순위
  중간(2): 검토 안건
  단일(1): 참고 (단발 시그널)
```

### §4. 평가 진단 (#5 상세)

평가자 단위 패턴 + 모드 명시:

```
모드: FALLBACK (전사 평가자 7명 — 풀 작음, 통계 신뢰도 제한적)
1차 필터 통과: 2명 (상위 1 + 하위 1)

평가자 E-021 (개발팀장):
  이번 평균 4.05 (1위) / 작년 (1위) / 재작년 (1위)
  → 패턴: 지속 상위
  → turnover 8% (낮음) → 평가자/팀 분리 어려움

평가자 E-007 (재무팀장):
  이번 평균 2.85 (7위) / 작년 (6위) / 재작년 (7위)
  → 패턴: 지속 하위
  → turnover 12% (낮음) → 분리 어려움
```

### §5. 한계 + 권고 액션 (HR 검토 우선순위)

```
[모드별 신뢰도]
- FULL (#1: G1~G3 / #5: 풀 20+):       정상 분석
- PARTIAL (#1: G4):                     신뢰도 제한적
- FALLBACK (#5 풀 7명):                 참고 자료 수준
- INSUFFICIENT (#1: G5 부장 4명):       표본 부족, 스킵

[법적 즉시 시정 — 우선순위 1]
- #9 법규 위반 1개 부서 (24명) → 즉시 시정 검토

[강 시그널 — 우선순위 2]
- §3 결합 강도 "강함" 사원 N명 → 면담 우선
- #7 과부하 부서 → 부담 분산 검토

[중 시그널 — 우선순위 3]
- §3 결합 강도 "중간" 사원 → 검토 안건
- #4 편중 부서 → #5 결합 후 평가자 패턴 확인

[참고 — 우선순위 낮음]
- §3 단발 시그널 → 단년 변동 가능성, 다음 시즌 추적
- #5 FALLBACK 결과 → 정성 검토 결합 필수
```

```
  - §1: 각 분석의 핵심 라벨·수치만 추출
  - §2: 부서 단위 라벨 매트릭스 구성
  - §3: 사원 후보 합집합 + 결합 강도 계산
  - §4: #5 결과 상세
  - §5: 모드별 신뢰도 + 우선순위 분류

3단계: 표준 템플릿 출력
  - 5개 섹션 고정 형식
  - AI 박스는 §1 의 1~2줄 + §3 강 시그널 사원당 1줄
```

## 사내 통상 비교 — 동적 임계치

본 보고서는 임계치를 새로 정의하지 않음. **하위 분석(#1~#9) 의 임계치를 그대로 사용** 하고 결과만 통합.

## 시그널 의미 — 통합 해석

| 시그널 | 의미 |
|--------|------|
| 부서가 §2 에서 여러 라벨 부착 | 다중 시그널 → 우선 검토 안건 |
| 사원이 §3 에서 결합 강도 강함 | 다중 분석 동시 도출 → 면담 우선 |
| §1 에서 변별력 약함 + §4 편중 + §5 지속 상위 | 보상·등급·평가 모두 의문 → 종합 점검 |
| §5 FALLBACK · §1 INSUFFICIENT 다수 | 회사 규모 작음 → 통계 신뢰도 한계, 정성 비중 ↑ |

→ 단정 X, 모든 시그널은 검토 후보 자료.

1. **법적 즉시 시정** — #9 법규 위반 부서·사원
2. **강 시그널** — §3 결합 강도 강함 사원, #7 과부하 부서
3. **중 시그널** — §3 결합 중간, #4 편중 부서
4. **참고** — 단발 시그널, FALLBACK 결과

## 표현 가이드 — 본문 vs AI 박스

본문 = 5개 섹션의 수치·라벨 (가치판단 X).

AI 참고 박스는 §1 끝과 §3 강 시그널 사원에만:

§1 종합 박스:
> "이번 시즌 전사 진단에서 법규 위반 1개 부서·강 시그널 사원 N명이 확인됩니다.
> 우선순위 1·2 안건은 즉시 검토를 권고하며, FALLBACK 모드 결과는 참고 자료로 활용하세요."

§3 사원별 박스 (강 시그널만):
> "EMP-023 은 보상 누락(#2)·부당 보상 후보 케이스B(#1-3)·안정 우수 패턴(#6) 3개 분석에서 동시 도출됐습니다.
> 즉시 면담을 통한 본인 의향·처우 재검토를 권고합니다."

## 한계

1. **하위 분석 한계 누적** — #1~#9 의 모든 한계를 그대로 상속. 본 보고서는 통합만 할 뿐 한계 보완 X
2. **부분 짬뽕 비효율** — 5섹션 고정 구조라 사용자가 2~3개만 원하면 본 도구 비효율. 그 경우 개별 도구 호출 권고
3. **결합 강도 단순 카운트** — §3 의 강도는 "도출된 분석 수" 만 셈. 시그널 강도의 정성 차이 미반영
4. **모드 다양성** — 회사 규모 작으면 #1·#5 가 INSUFFICIENT/FALLBACK 모드로 가서 보고서 가치 감소. 본문에 모드 명시 필수
5. **자유 쿼리 대체 X** — "L4 직급 우수 사원 보여줘" 같은 단발 질문은 본 도구가 부적합 (개별 SQL 쿼리 또는 #1 직접 호출)
6. **분기·반기 단위 적합** — 매주·매일 호출은 비용 대비 효율 낮음. 시즌 종료 후 1~2회 호출 권고
